=======================================================
PHASE 2: GOLF IT DOWN
=======================================================

Crush the clean version into the fewest possible characters.
Save it to Day-NN/JavaScript/tmp/golf.js.

Use every JavaScript golf trick you know:

INPUT/OUTPUT TRICKS
  - Shortest stdin read for Node.js
  - Use console.log or process.stdout.write, whichever is shorter
  - Template literals to avoid string concatenation

ARROW FUNCTIONS & EXPRESSIONS
  - Replace function declarations with arrow functions: x=>x*2
  - Single-expression arrows need no braces or return
  - Use comma operator to sequence side effects in an expression:
      (a=1, b=2, a+b) instead of { a=1; b=2; return a+b; }
  - Immediately invoked arrows: (f=>f(f))(f=>...)

VARIABLE & NAMING TRICKS
  - Single-character variable names everywhere
  - Destructuring assignment to declare multiple vars concisely:
      [a,b,c]=[1,2,3]
  - Reuse loop variables, abuse var hoisting
  - Omit let/const/var entirely when sloppy mode allows it

LOOP COMPRESSION
  - for(;;) with comma-packed body instead of while
  - for...of is often shorter than indexing: for(x of a)
  - Replace loops with .map/.reduce/.filter/.flatMap chains
  - .reduce can replace almost any loop
  - Array.from({length:n},(_,i)=>...) for range generation
  - Recursive solutions when they save bytes

TYPE COERCION ABUSE
  - +x to convert to number instead of parseInt/Number
  - x+'' to convert to string instead of String(x)
  - ~~x for Math.floor on positives
  - x|0 for integer truncation
  - !x for boolean, !!x for strict boolean
  - -(-x) or x- -y to avoid ambiguous + operators
  - [] and {} in boolean context

OPERATOR TRICKS
  - || and && for short-circuit defaults and conditionals
  - ?? (nullish coalescing) when it saves over ||
  - ?. (optional chaining) to avoid existence checks
  - ** instead of Math.pow
  - Bitwise: &, |, ^, ~, <<, >> for math tricks
  - Spread ...x instead of Array.from or .concat

STRING & REGEX
  - .split`` or .split`\n` (tagged template = split on that string)
  - /regex/.test(x) or x.match(/regex/) depending on what's shorter
  - .replace with callback for transform-and-extract patterns
  - Template literals for multi-line strings and interpolation

=======================================================
PHASE 3: EXPAND WITH OBFUSCATION
=======================================================

Take the golfed core and EXPAND it — not back toward clarity, but
deeper into darkness. Longer AND more confusing. Save it to
Day-NN/JavaScript/tmp/obfusc.js.

Layer these techniques, choosing the ones that work best together:

TYPE COERCION SORCERY
  - Construct strings from coercion:
      (![]+'')[0]            → 'f' (from "false")
      (![]+[])[+!![]]        → 'a'
      ([]+{})[+!![]]         → 'o' (from "[object Object]")
  - Build function names character by character from coerced values,
    then invoke them via []['constructor']['constructor']('...')()
  - Use +[] for 0, +!![] for 1, and build all other numbers from these
  - JSFuck-style encoding for critical strings (don't encode EVERYTHING
    this way or it becomes unreadable noise — use it selectively for
    key strings while leaving other parts conventionally obfuscated)

ENCODING & DATA HIDING
  - Encode strings as arrays of char codes:
      [72,101,108,108,111].map(x=>String.fromCharCode(x)).join('')
  - XOR cipher: encode a string against a key, decode at runtime
  - Base64 via Buffer.from('...','base64').toString() in Node
  - Store data in unusual places: environment variables, the filename
    itself (process.argv[1]), or computed from Date/Math constants
  - Hide lookup tables in long template literal strings, index into
    them at runtime

CONTROL FLOW POISONING
  - Replace all control flow with nested ternary expressions:
      if/else → a ? b : c ? d : e ? f : g
  - Replace loops with recursive self-invoking arrow functions
  - Use .reduce() chains where the accumulator is a function, not data
  - Promises and async/await used unnecessarily to obscure sequencing
  - setTimeout/setImmediate to scatter execution order
  - try/catch/finally where the "error" path is the real logic and
    the "success" path is dead code

PROTOTYPE MANIPULATION
  - Override Array.prototype or String.prototype methods so built-in
    names do unexpected things
  - Add custom methods to Number.prototype or Boolean.prototype
  - Use Proxy objects to intercept property access and perform
    computation in the get/set traps
  - Object.defineProperty with getters that have side effects
  - Symbol.toPrimitive to make objects that behave differently in
    different type contexts

MISDIRECTION
  - Variables named to imply wrong purpose (total that holds a string,
    output that's never logged, error that contains the answer)
  - Dead code paths that look critical (complex calculations whose
    results are thrown away)
  - eval() calls that look dangerous but just compute a number
  - Regular expressions that look like they're matching patterns but
    are actually used for their side effects in .replace callbacks
  - console.log buried inside expressions that also do real work:
      (console.log(x), realComputation(x))

SCOPE & CLOSURE TRICKS
  - Self-invoking functions that capture variables in confusing
    closure chains
  - with() statements (sloppy mode) to pollute scope lookup
  - Function constructor: new Function('a','b','return a+b')
  - eval() to introduce variables into unexpected scopes
  - arguments object abuse, callee references (sloppy mode)

Show the code and confirm correct output for both example and full
