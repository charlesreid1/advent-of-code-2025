WHAT IT DOES
------------
This program solves both parts of Advent of Code 2025 Day 3 ("Lobby").
It reads battery joltage ratings from stdin, one bank per line, and computes:

Part 1: The maximum 2-digit joltage obtainable from each bank
Part 2: The maximum 12-digit joltage obtainable from each bank

The program outputs the sum for part 1, then the sum for part 2.

OBFUSCATION TECHNIQUES
----------------------
1. **Visual confusion**: Variable names use visually similar characters:
   - O (capital O), 0 (zero), o (lowercase o)
   - l (lowercase L), 1 (one), I (capital i)
   - _ (underscore), __ (double), ___ (triple), ____ (quadruple)

2. **Rectangular shape**: The source code forms a perfect rectangle of
   comment blocks, with the actual code woven between them.

3. **Arithmetic obfuscation**: Constants are derived from character
   arithmetic rather than literal numbers.

4. **Dense formatting**: Minimal whitespace, with logical breaks occurring
   at seemingly arbitrary points in the middle of identifiers.

5. **Misdirection**: The comment blocks filled with 'M' characters serve
   no purpose other than to complete the rectangular shape and distract
   from the actual algorithm.

ALGORITHM
---------
Despite appearances, the algorithm is straightforward:

- For part 1: Exhaustively check all pairs of digits in each line.
- For part 2: Greedy selection of maximum digits while maintaining
  enough remaining positions to complete the 12-digit number.

The clever part is how these algorithms are hidden within the visual
noise of similar-looking variable names and the rectangular comment
scaffolding.

-------------------
Compile with: make

Run with: ./solution < input.txt

Test with: make test

The program expects input on stdin, one battery bank per line.
Each line should contain only digits 0-9.
