PHASE 2: GOLF IT DOWN
=======================================================

Crush the clean version into the fewest possible characters.
Save it to Day-NN/C/tmp/golf.c.

Use every C golf trick you know:

INPUT/OUTPUT TRICKS
  - scanf/printf with minimal format strings
  - getchar() loops instead of fgets/fread
  - putchar() instead of printf for single characters
  - Read until EOF: while(~scanf(...)) or while(gets(s))
  - Use return value of scanf as loop condition

VARIABLE & TYPE TRICKS
  - Single-character variable names everywhere
  - Implicit int for main (compile as C89/gnu99 if needed)
  - Global variables to avoid declarations (default-initialized to 0)
  - Reuse variables for multiple purposes across the program
  - char* instead of arrays where possible
  - Comma-separated declarations: int a,b,c,d;

CONTROL FLOW COMPRESSION
  - Ternary chains instead of if/else: a?b:c?d:e
  - Short-circuit for conditional execution: a&&f(); instead of if(a)f();
  - Comma operator to pack statements: for(a=0;a<n;a++,b+=c,d--)
  - Collapse nested loops into a single loop with computed indices
  - Combine loop body into the for() increment expression, leaving
    the body empty: for(...;...;body1,body2,body3);

EXPRESSION TRICKS
  - Bitwise: & | ^ ~ << >> for arithmetic tricks
  - !!x for boolean conversion
  - Array indexing tricks: "FT"[condition] instead of condition?"T":"F"
  - Pointer arithmetic instead of array indexing
  - XOR swap, XOR tricks for conditional flipping
  - Abuse sizeof for constants: sizeof(int) instead of 4
  - a[b] is the same as b[a] is the same as *(a+b)

PREPROCESSOR TRICKS
  - #define short aliases for repeated patterns
  - Macro for main boilerplate: #define F for(
  - But don't over-macro — the golf version should be readable
    enough to verify correctness before obfuscation

=======================================================
PHASE 3: OBFUSCATION
=======================================================

Take the golfed core and EXPAND it with obfuscation. The code should
grow longer AND more confusing. Save to Day-NN/C/tmp/obfusc.c.

Apply these families of techniques. You do NOT need to use all of
them — choose the ones that work best together for THIS program.

STRUCTURAL OBFUSCATION
  a) Merge all functions into main() using goto or longjmp.
  b) Replace remaining control flow with ternary operator, comma
     operator, or short-circuit evaluation (&&, ||).
  c) If you kept a single loop from the golf phase, make its
     structure confusing — use for(;;) with breaks triggered by
     computed conditions, or replace while() with recursive main().
  d) Replace if/else trees with arithmetic expressions:
       if(x>0) a=1; else a=2;  →  a=2-(x>0);
  e) Convert recursion to iteration (or vice versa) in a way that
     hides the algorithm.

DATA & IDENTIFIER OBFUSCATION
  a) Rename every variable to visually confusing names. Good sets:
       O 0 o (letter O, zero, lowercase o)
       l 1 I (lowercase L, one, uppercase I)
       _ __ ___ (underscores of varying length)
       Mix: O0, l1, Il, _o, O_, etc.
  b) Encode string literals as:
       - Integer arrays decoded at runtime
       - XOR-encoded blobs with a key
       - Multi-character constants: 'ABCD'
       - Arithmetic on pointers into existing strings
  c) Replace numeric constants with opaque expressions:
       42   → (6*9-12+6)
       0    → (sizeof(char)-1)
       1    → (sizeof(char))
       10   → ('\n')
       Derive constants from sizeof, pointer differences, or
       character arithmetic: 'z'-'a'+1 for 26
  d) Use #define macros to alias keywords:
       #define _ for
       #define __ if
       #define ___ while
       #define O return
     But use them SPARINGLY and make the aliases confusing,
     not self-documenting.

PREPROCESSOR & TYPE ABUSE
  a) Create macros that expand into partial statements or
     unbalanced braces:
       #define B(x) {x;{
       #define E }}
  b) Use digraphs if they add confusion:
       <: for [    :> for ]
       <% for {    %> for }
  c) Abuse old-style K&R function declarations.
  d) Cast function pointers through void* and back.
  e) Use the stringizing (#) and token-pasting (##) operators
     to generate identifiers or strings from macro arguments.

MISDIRECTION
  a) Dead code that looks important (complex calculations whose
     results are never used).
  b) Variables whose names imply wrong types or purposes.
  c) Comments that lie: /* this is the base case */ before code
     that is definitely not the base case.
  d) Macros whose names suggest one operation but perform another:
       #define SUM(a,b) ((a)*(b))

Show the code and confirm correct output for both example and full
input before proceeding.
