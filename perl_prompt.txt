PHASE 2: GOLF IT DOWN
=======================================================

Crush the clean version into the fewest possible characters.
Save it to Day-NN/Perl/tmp/golf.pl.

Use every Perl golf trick you know:

  - Implicit $_ everywhere
  - -p/-n flag abuse in the shebang line
  - say instead of print when possible
  - Collapse loops into map/grep
  - Use || and && for control flow instead of if/else
  - Ternary chains instead of conditionals
  - String repetition (x) and list repetition
  - Regex as computation, not just matching
  - Collapse multiple statements with comma operator
  - Drop my declarations, let variables be global
  - Use qw// and other quote-like operators to save delimiters
  - Chained chops, pops, shifts

  - Single-character variable names throughout

=======================================================
PHASE 3: EXPAND WITH OBFUSCATION
=======================================================

Take the golfed core and EXPAND it — not back toward clarity, but
deeper into darkness. Longer AND more confusing. Save it to
Day-NN/Perl/tmp/obfusc.pl.

Layer these techniques, choosing the ones that work best together:

ENCODING & DATA HIDING
  - Replace string literals with pack/unpack, chr/ord chains,
    v-strings, XOR ciphers, or base conversion
  - Encode numbers as expressions: 42 → unpack("C","*")
  - Store data in unexpected places: $0, $ENV{}, __DATA__,
    the source code itself read via __FILE__
  - Use heredocs that contain encoded data the program decodes

CONTROL FLOW POISONING
  - Replace loops with recursive anonymous subs via goto &$sub
  - Hide flow in eval'd constructed strings: eval join'',map{...}
  - Use BEGIN/END/AUTOLOAD/DESTROY to scatter execution order
  - Dispatch tables (hashes of coderefs) instead of conditionals
  - Short-circuit chains that go five or six levels deep

PERL BLACK MAGIC
  - Tie a variable to a class so FETCH/STORE does real work
  - Overload operators on a blessed ref so math produces output
  - AUTOLOAD that intercepts non-existent method names as data
  - Source filters that rewrite code at compile time
  - Use formats (format/write) for output, hiding logic in
    picture lines

MISDIRECTION
  - Variables named to imply wrong purpose ($counter that holds
    a string, @output that's never printed, %important that's empty)
  - Code paths that look critical but are dead
  - Operators that look like variable names and vice versa
  - Comments that lie

SPECIAL VARIABLE ABUSE
  - $/ $\ $, $; $" to change I/O behavior implicitly
  - $_ as silent argument to everything
  - $! $@ for their string/numeric duality
  - $[ if feeling truly evil
  - Regex match vars $& $` $' $1 etc. as data storage

Show the code and confirm correct output for both example and full
input before proceeding.
